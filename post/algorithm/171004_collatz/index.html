문제: 콜라츠 추측

1937년 Collatz란 사람에 의해 제기된 이 추측은, 입력된 수가 짝수라면 2로 나누고, 홀수라면 3을 곱하고 1을 더한 다음, 결과로 나온 수에 같은 작업을 1이 될 때까지 반복할 경우 모든 수가 1이 된다는 추측입니다. 예를 들어, 입력된 수가 6이라면 6→3→10→5→16→8→4→2→1 이 되어 총 8번 만에 1이 됩니다. collatz 함수를 만들어 입력된 수가 몇 번 만에 1이 되는지 반환해 주세요. 단, 500번을 반복해도 1이 되지 않는다면 –1을 반환해 주세요.

풀이

  • while문의 조건으로 반복횟수와 결과 체크
  • 매 반복마다 횟수(result) 1씩 누적. 초기값은 1.

딱히 생각할 게 없어서.. 일단 작성

def collatz(num):
    result = 1
    while num / 2 != 1 and result != 500:
        result += 1
        if num % 2 == 0:
            num = num / 2
        else:
            num = num * 3 + 1
    return result

틀렸다(!) 500회 반복 시 -1 반환하는 것을 빼먹었다. 따라서 반복문 밖에 result가 500일 때 -1로 재할당하는 구문을 추가한다. 500번만에 딱 1이 되는 경우도 있을 테니 2로 나눠서 1이 되는지도 체크한다.

def collatz(num):
    result = 1
    while num / 2 != 1 and result != 500:
        result += 1
        if num % 2 == 0:
            num = num / 2
        else:
            num = num * 3 + 1
    if num / 2 != 1 and result == 500:
        result = -1
    return result

자바스크립트로도 풀어보자.

  1. 초기값을 500으로 한 while문을 사용.
    while(i--) 다만 이 방법은 반복수를 따로 계산해줘야 하는 불편함이 있다.
  2. while(i < 500) 을 사용하면 반복문 탈출 시점의 i를 재활용 할 수 있다.

2번 방법에 break를 추가해서 함수를 작성한다.

function collatz(num) {
  var answer = 0;
  while (answer < 500) {
    answer += 1;
    if (num / 2 == 1) {
      break;
    } else if (num % 2) {
      num = num * 3 + 1;
    } else {
      num = num / 2;
    }
  }
  return answer;
}

아… 이번에도 500 처리를 빼먹었다. 반복문 밖에서 500을 -1로 바꾸는 구문을 추가한다.

function collatz(num) {
  var answer = 0;
  while (answer < 500) {
    answer += 1;
    if (num / 2 == 1) {
      break;
    } else if (num % 2) {
      num = num * 3 + 1;
    } else {
      num = num / 2;
    }
  }
  if (answer == 500) {
    answer = -1;
  }
  return answer;
}

문제를 똑바로 읽기. 조건을 모두 만족하는 지 꼼꼼히 체크. 반복문 조건식을 유연하게 사용하기.(while(i--) 그만…) iteration 변수를 블록 내부에서 활용할 수 있도록 설계하기. 끝.
